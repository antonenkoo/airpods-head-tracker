# AirPods Head Tracker (нативно, macOS) 🎧

Реальная ориентация головы **с AirPods Pro 2** (а также Pro 1 / Max / AirPods 3+).
Нативное Swift-приложение с окном: читает движения через `CMHeadphoneMotionManager`,
интерфейс (3D-куб, монитор осанки, управление музыкой) показывается в WKWebView.

> Из браузера напрямую это невозможно — `CMHeadphoneMotionManager` доступен только
> нативным приложениям. Работает на **macOS 14+**.

## Сборка и запуск

```bash
cd airpods-native
./build.sh          # компилирует AirPodsTracker.app (Xcode не нужен, хватает CLT)
open AirPodsTracker.app   # окно с интерфейсом откроется само
./make-dmg.sh       # упаковать в DMG для распространения
```

Тот же интерфейс параллельно доступен в браузере: **http://localhost:8765**
(удобно открыть с телефона в той же сети).

Надень AirPods, посмотри прямо, нажми **«Калибровать»** — и крути головой.

### Первый запуск: разрешение
При первом обращении к датчику macOS спросит доступ к движению — **разреши**.
Если приложение запущено через `open …app`, логи уходят в Console.app.
Чтобы видеть логи прямо в терминале, запусти бинарь напрямую:

```bash
./run.sh
```
(в этом режиме TCC-запрос иногда не всплывает — тогда один раз запусти через `open AirPodsTracker.app`,
выдай разрешение, дальше можно из терминала.)

## Проверка без наушников
```bash
curl http://localhost:8765/orientation
# {"yaw":0.00,"pitch":0.00,"roll":0.00,"connected":false}
```
`connected` станет `true`, как только пойдут данные с AirPods.

## Эндпоинты
- `GET /`            — страница-визуализация
- `GET /orientation` — текущие углы в градусах: `{yaw,pitch,roll,connected}`

## Порт
`PORT=9000 ./run.sh` — сменить порт (по умолчанию 8765).

## Если данных нет (`connected:false` даже в наушниках)
- AirPods должны быть **в ушах** и активны как звуковой выход этого Mac.
- Модель должна поддерживать Spatial Audio с трекингом головы (Pro 1/2, Max, AirPods 3+).
- Проверь разрешение: Системные настройки → Конфиденциальность → **Движение и фитнес**
  (или Bluetooth) → AirPodsTracker включён.

## Файлы
```
airpods-native/
├── main.swift      # CoreMotion + HTTP-сервер + запуск NSApplication
├── app.swift       # окно приложения (NSWindow + WKWebView, меню)
├── html.swift      # встроенный интерфейс (единый для окна и браузера)
├── gen-icon.swift  # генератор иконки (CoreGraphics → PNG → icns)
├── build.sh        # сборка .app + Info.plist + ad-hoc подпись
├── make-dmg.sh     # упаковка в DMG
└── run.sh          # прямой запуск (логи в терминал)
```
